================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

COMPONENT INTEGRATION:
  • UMLS API Client → Ontology Tool → Agent Engine
  • Automatic retry with exponential backoff (429 rate limits)
  • Connection pooling via requests.Session
  • Single shared client instance per process

TOOL SPECIFICATION:
  • Name: query_umls_ontology
  • Argument: term="medical_term" (string)
  • Returns: ToolResult with CUI, relationships, summary, error
  • Two-step process: search_concept → get_concept_relations

ERROR HANDLING:
  ✅ Rate limiting (429) - automatic retry with backoff
  ✅ Network timeouts - retry up to 3 times
  ✅ Connection errors - transparent retry logic
  ✅ Missing API key - clear error message with setup instructions
  ✅ Concept not found - graceful failure with error details

SAFETY CONSTRAINTS ENFORCED:
  "If a medical relationship is not explicitly verified by the 
   query_umls_ontology or query_medical_ontology tool, you MUST 
   state that you cannot confirm the clinical relationship."

USAGE EXAMPLES
================================================================================

STEP 1: Get API Key
  → Visit https://uts.nlm.nih.gov/uts/
  → Sign up → My Profile → API Key

STEP 2: Set Environment
  export UMLS_API_KEY='your-key-here'

STEP 3: Test It
  python demo_umls_ontology.py

STEP 4: Use in Code

  FROM DIRECT CLIENT:
    from umls_client import search_concept, get_concept_relations
    cui = search_concept("Lutetium Lu 177 dotatate")
    relations = get_concept_relations(cui)

  FROM WRAPPER:
    from ontology_tool import verify_clinical_relationship
    result = verify_clinical_relationship("Lutetium Lu 177 dotatate")
    if result.found:
        print(result.summary)

  AGENT AUTOMATICALLY CALLS:
    ACTION: query_umls_ontology(term="Lutetium Lu 177 dotatate")
    → Returns verified relationships from UMLS
    → LLM uses for response generation
